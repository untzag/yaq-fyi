yaq/traits/has-dependents

Daemons with this trait control or otherwise depend on the existence of other running yaq daemons

messages ¶

Daemons with the has-dependents trait expose the following public messages:

get_dependent_hardware ➜ {'type': 'map', 'values': 'string'}
Return a map of names to host:port strings for dependent yaq daemons Clients are expected to translate localhost/127.0.0.1 to the host on which they contact the parent. Clients are not guaranteed to be able to communicate with dependent hardware, even if the parent can.

daemons ¶

attune
attune-delay
ndinterp
